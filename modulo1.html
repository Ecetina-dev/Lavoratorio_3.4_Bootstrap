La ingeniería de software requiere un enfoque sistemático, disciplinado y cuantificable. Existen 4 descripciones fundamentales que no deben saltarse para evitar costos exponenciales: Requerimientos (R), Especificación (S), Diseño (D) y Programa (P). Para gestionar el ciclo de vida del proyecto, se pueden utilizar modelos como el de Cascada o métodos Ágiles como Scrum.

Los requerimientos son guías que dictan qué hace el sistema y sus restricciones. Para priorizarlos, se utiliza el método DINO: Deseables, No implementables, Importantes y Obligatorios. Es crucial recordar que los requerimientos No Funcionales (seguridad, desempeño) son siempre Obligatorios.

La arquitectura vive en la etapa de Diseño, ubicándose entre la Especificación y el Código. Se divide en tres niveles de estructura: Estilo Arquitectónico, Patrones de Diseño y Diseño de Componentes. El objetivo supremo no es solo el orden, sino permitir modificar el sistema a bajo costo en el futuro.

Un patrón de diseño es una solución probada a un problema recurrente; sirve como un molde universal adaptable a cualquier lenguaje. Un ejemplo destacado es el patrón Singleton, cuyo objetivo es garantizar que una clase tenga una única instancia en toda la aplicación.

Ejemplo de Código (Singleton en Java): 

public class Conexion {
    private static Conexion instancia;
    
    private Conexion() {}
    
    public static Conexion getInstancia() {
        if(instancia == null) {
            instancia = new Conexion();
        }
        return instancia;
    }
}
