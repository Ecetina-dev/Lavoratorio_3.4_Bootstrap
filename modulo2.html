La Programación Orientada a Objetos (POO) es un paradigma de diseño de software que modela conceptos del mundo real mediante "clases" y "objetos". Se basa en cuatro pilares fundamentales: encapsulamiento, abstracción, herencia y polimorfismo.

Este tema integra los conocimientos de POO y bases de datos mediante la construcción de un sistema real. Se implementa la conexión directa desde el código Java hacia el motor de MySQL utilizando controladores JDBC. Se aplica el patrón de arquitectura MVC para separar la interfaz gráfica, la lógica de negocio y el acceso a los datos, culminando en un sistema con operaciones CRUD completas.
